## Common Questions & Answers

### "Why focus on UX first (Morning Routine) instead of repricing?"
**Answer:** Because 80% of problem is user confusion. If you fix that first, repricing gets adopted faster. They're not sequential—they run in parallel (Week 1-2 UX, Week 2-3 repricing).

### "Is 5 days for repricing automaton realistic?"
**Answer:** Yes, with caveats. Backend code exists (suggestions work). We're adding automation layer (scheduler) + safety rules. 5 days assumes good tests + no blocking issues.

### "What if repricing breaks and loses sales?"
**Answer:** Guardrails prevent this: max 20% increase/day, min margin protection, disables if competitor moved price. Testing starts with 3-5 ads only.

### "Who's responsible for what?"
**Answer:** Frontend team = UX/design. Backend team = automation/logic. Maikeo = validation/feedback. QA = testing. PM = coordination.

### "When do we know Phase 1 is successful?"
**Answer:** When Maikeo says "Now I understand what to do" and uses repricing 10+ times in April.
